# MVP Roadmap

## Stage 1: Product Skeleton

- API structure and DB schema.
- Dashboard shell.
- Public storefront shell.
- Docker Compose infrastructure.

## Stage 2: Core Commerce

- Registration/login.
- Store onboarding.
- Product CRUD.
- Category CRUD.
- Public catalog.
- Guest checkout.
- Order CRM.

## Stage 3: AI Layer

- Store generation.
- Product descriptions.
- SEO generation.
- AI usage limits by tariff.
- `ai_generations` logging with provider, token and cost fields.

## Stage 4: Telegram

- Bot webhook.
- Store owner Telegram linking.
- New order notifications.
- Daily store summary.

## Stage 5: Admin and Billing

- Platform admin users/stores view.
- Tariff management.
- AI limits.
- Basic analytics: stores, orders, GMV, MRR placeholder.

## MVP Acceptance

- Seller creates a store in under 5 minutes.
- Seller adds a product and generates AI text.
- Public store works on mobile.
- Buyer places an order without registration.
- Seller sees the order in CRM and receives Telegram notification.
- Admin sees users and stores.
